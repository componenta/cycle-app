# Componenta Cycle App

Интеграционный пакет для `componenta/cycle` в Componenta-приложении. Он связывает Cycle-сервисы с обнаружением классов, скомпилированной конфигурацией и консольными командами.

Используйте его в приложении, которому нужно управляемое фреймворком обнаружение Cycle-сущностей. Библиотеки должны зависеть только от `componenta/cycle`.

## Установка

```bash
composer require componenta/cycle-app
```

Регистрируйте провайдер после рантайм-провайдера Cycle:

```php
return [
    new Componenta\Cycle\ConfigProvider(),
    new Componenta\Cycle\App\ConfigProvider(),
];
```

## Связанные пакеты

| Пакет | Зачем нужен здесь |
|---|---|
| `componenta/cycle` | Содержит репозитории, fetcher-классы, фильтры, typecasts и фабрики времени выполнения. |
| `componenta/class-finder` | Находит сущности и embeddable-классы в настроенных директориях. |
| `componenta/app` | Запускает сборку кеша и выбирает режим разработки или продакшена. |
| `cycle/orm` | Потребляет итоговую ORM-конфигурацию. |

## Что добавляет

- обнаружение сущностей и embeddable-классов;
- locator-сервисы, основанные на настроенном итераторе классов;
- записи `ClassFinderConfigKey::LISTENERS` для `EntityLocator` и `EmbeddingLocator`;
- консольные команды Cycle;
- интеграцию кеша и компиляторов для сборки приложения.

## Режим разработки

В разработке приложение может сканировать настроенные директории исходников и выводить Cycle mappings из найденных классов. Это позволяет держать конфигурацию модуля сфокусированной на классах, которыми владеет модуль.

## Продакшен

В продакшене приложение должно использовать скомпилированную конфигурацию и сгенерированные файлы кеша. Оно не должно сканировать исходники или пересобирать ORM-метаданные на каждый запрос.

## Границы

Этот пакет не должен содержать поведение работы с базой, нужное рантайм-потребителю напрямую. Репозитории, data fetchers, фильтры, typecasts и фабрики времени выполнения принадлежат `componenta/cycle`.
